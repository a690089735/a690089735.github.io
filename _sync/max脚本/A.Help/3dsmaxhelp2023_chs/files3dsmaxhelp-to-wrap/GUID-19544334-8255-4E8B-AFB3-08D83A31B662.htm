资源追踪

通过“资源追踪”功能，可以在 3ds Max 中直接访问 Asset Tracking System (ATS)，也称之为提供者。使用资源追踪系统，可以与开发团队的其他成员共享材质中所使用的场景文件和位图文件等。

使用资源跟踪器管理路径

“资源追踪”对话框的另一个重要功能就是重新指定路径并查找丢失的文件。例如，如果将场景中材质所使用的位图文件移至与场景文件相同的文件夹中，则在打开场景文件时，会加载位图文件，但材质将仍然使用位图文件原始的、不再有效的路径，“资源跟踪”对话框将这些文件显示为丢失的文件。要解析不正确的路径，可以使用“路径”菜单上提供的工具。

使用 Autodesk 库的资源追踪

资源追踪完全支持 Autodesk 库数据管理解决方案，并为其他的提供程序提供基本的版本控制，如 Perforce 和 Microsoft SourceSafe。通常，资源追踪支持能集成到 Microsoft Visual Studio 中的版本控制提供程序，有时也称为 MSSCC 支持。本主题假设使用 Autodesk 库。对于固定期限的使用许可客户，Autodesk Data Management Server 和 Autodesk Vault Explorer 均可用，并且附带到某些其他 Autodesk 软件。Vault 插件附带在 3ds Max 中。

注意:安装 Vault 之后，可以使用“文件”菜单命令“从 Vault 打开”，直接从 Vault 数据库中打开文件。该命令与文件打开过程很相似，但它浏览库而不是文件系统。除了打开场景文件之外，它还可以下载或更新任何从属场景文件，如位图和外部参照。如果已安装 Vault 插件，则“文件”菜单中将显示“从 Vault 打开”。

通过向界面添加数据管理工具，3ds Max Vault 插件可以与 3ds Max 协同工作。通过 Vault 插件，您可以将文件添加到 Vault，然后对文件进行检入和检出。插件可与很多类型的文件一起使用，其中包括 MAX 和图像文件。可根据您的工作环境执行库操作的推荐方法。

当您处理检出库的文件时，您可以处理本地文件副本而不是原始文件。您处理的不是实际入库的文件。当您将修改的文件检入到库中时，修改同样适用于库中的最新版本。库中保留文件的所有旧版本。
