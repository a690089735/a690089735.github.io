“mParticles 世界”操作符

“mParticles 世界”操作符将当前事件或系统中的粒子关联到特定模拟世界，所用形式为场景中的“mParticles 世界”辅助对象（驱动程序）。完成连接后，该辅助对象的设置会影响事件中的所有粒子。

  • 粒子视图 添加或选择“mP 世界”。

可以将“mParticles 世界”操作符放置于局部事件和全局事件。事件中操作符的存在表示该事件中的粒子参与物理模拟。如果此操作符位于全局事件中，则粒子流系统中的所有粒子始终参与模拟。

事件中的粒子只能属于一个模拟世界，相应粒子由事件中的“mParticles 世界”操作符指定。虽然可以将多个“mParticles 世界”操作符添加至事件，但是该操作会产生歧义，导致粒子流会将其全部忽略。

事件中“mParticles 世界”操作符（带有关联的“mParticles 世界”驱动程序）的存在不足以启动模拟。第二个必需的组件是用于定义粒子模拟属性的“mParticles 图形”操作符。始终在事件中的“mParticles 世界”操作符前放置“mParticles 图形”操作符，因此针对模拟，“mParticles 世界”操作符将了解由 mParticles 图形定义的粒子属性。

注意:不同粒子流系统中的粒子（具有不同的全局粒子流源事件）无法参与相同的模拟世界。这是由于粒子是通过模拟世界中的粒子 ID 来追踪的，而每个粒子流源创建自己的 ID 集合。即使两个不同粒子流系统中的“mParticles 世界”操作符关联同一“mParticles 世界”驱动程序，并且创建两个模拟世界（每个粒子流系统一个世界），但两个模拟具有“mParticles 世界”驱动程序定义的相同全局属性。

[无]
若要指定影响当前事件中的粒子的“mParticles 世界”辅助对象，请单击此按钮，然后选择辅助对象。
=>
在标准命令面板打开关联的“mParticles 世界”驱动程序的参数。
创建新的驱动程序
在场景中自动创建新的“mParticles 世界”辅助对象，并使“mParticles 世界”操作符关联该辅助对象。
取消快速保存
粒子流具有已集成的“快速保存”功能，该功能允许当前帧不为 0 时进行快速保存。但是，此功能可能会干扰 MassFX 模拟的完整性。因此，如果您拥有复杂的 MassFX 设置，并且模拟在第 0 帧时或在第 0 帧之前开始，则建议启用“取消快速保存”（默认为启用状态）。这时，已集成的“快速保存”功能在具有“mParticles 世界”操作符的事件中禁用。
