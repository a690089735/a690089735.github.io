“另存为”对话框 (Biped)

“另存为”对话框可以用来保存 Biped (BIP)、体形 (FIG) 和步长 (STP) 文件。

  • 选择 Biped。 “运动”面板 “Biped”卷展栏 （保存文件）

当保存 FIG 文件时（在体形模式），“另存为”对话框是一个标准的 Windows“保存文件”对话框。然而，当保存 BIP 或 STP 文件时，可以将对象和动画控制与该文件一起保存。

界面

FIG、BIP 或 STP 文件的“另存为”对话框

历史记录
此下拉列表用于选择您已经保存的 Biped 运动文件所在的目录。请参阅“运动文件历史纪录”列表。
查找范围
打开一个导航窗口以移动至其他目录或驱动器。
转至上次访问的文件夹
单击可以返回上次浏览的文件夹。
向上一级
在目录结构中向上移动一级。
创建新的文件夹
用于在该对话框中新建文件夹。
视图菜单
可以选择要在文件列表中显示的详细信息级别。
[列表窗口]

当选择“详细信息”时，目录所显示的内容包括名称、大小、类型、修改日期以及属性。可以通过单击每个参数的标签来对文件进行排序。

文件名
FIG、BIP 或 STP 文件的名称。
另存为类型
为保存的文件设置文件类型。
  • 体形文件 (.FIG)在体形模式中保存 Biped 的结构和位置。在体形模式中，当将 Biped 适配到网格后，保存一个体形文件。在“体形”模式中，如果 Biped 发生了意外的移动，重新加载此文件。
  • Biped 文件 (.BIP)保存 Biped 运动文件。BIP 文件包含了足迹和关键帧数据。Biped 文件存储了完整的运动并可以创建运动库。通过制作 Biped 动画并保存为 BIP 文件，可以创建自己的 BIP 库。
  • 步长文件 (.STP)以 ASCII 的格式保存足迹计时和位置数据。步长文件不包含躯干关键点（脊椎、手臂等等）。
[加号按钮]
以包含递增编号的文件名保存文件。不覆盖现有文件。
在当前位置和旋转上保存分段
选择 BIP 或 STP 文件的动画分段。
  • 从：从已命名动画的此帧开始保存。
  • 到：在已命名动画的此帧停止保存。
  • 每帧一个关键帧在每一帧上保存一个关键点。如果要使用“运动捕捉”卷展栏上的“加载运动捕获”来提取足迹，打开此项。
  • 活动时间段：启用该选项后，将保存活动时间段中的动画。
  • 编辑的变换间隔：仅保存变换中的帧。要激活此按钮，请在“运动流”卷展栏 “脚本”列表中选择一个剪辑，打开过渡编辑器，然后在“另存为”对话框中单击“保存文件”，启用“在当前位置和旋转上保存分段”。
    提示在游戏开发中，仅保存过渡中的运动十分实用。
  • Biped 动画周期：选中该选项之后，只保存包含 Biped 动画的时间。

“保存 MAX 对象”组

保存 MAX 对象
启用此选项后，将场景中的任何 IK、“头部目标”或链接对象保存为 BIP 文件。
[对象列表]
在此显示与场景中 Biped 相关联的每个对象及其对象类型。
  • 全部选中此选项后，所有关联对象都会与 BIP 文件一起保存。
  • 选定选中此选项后，只有高亮显示的对象会与 BIP 文件一起保存。
[选择对象按钮]
单击以打开一个对话框，可以在其中选择要与文件一起保存的其他对象。此对话框上的控件与场景资源管理器控件相同。

“保存列表控制器”组

保存列表控制器
启用此选项时，将任何指定到 Biped 的运动控制器都保存为 BIP 文件。
[控制器列表]
在此显示场景中 Biped 所使用的每个动画控制器及其控制的对象。
  • 全部启用此选项后，将所有关联动画控制器都保存为 BIP 文件。
  • 选定启用此选项后，将只有高亮显示的动画控制器保存为 BIP 文件。
